Halaman ini menjelaskan format report yang kami kirimkan ke URL Anda jika menggunakan transaksi via IP (untuk Otomax, IRS, dll).

> **Penting:** Pastikan IP kami sudah di-whitelist di server Anda: `188.166.243.56`

---

## Format Report

Report dikirimkan via GET request ke URL Anda dengan parameter berikut:

### Parameter

| Parameter  | Deskripsi                                      |
|------------|------------------------------------------------|
| serverid   | ID transaksi dari Toko Voucher                 |
| clientid   | ID referensi dari Anda                         |
| statuscode | Status transaksi: `1` = sukses, `2` = gagal    |
| kp         | Kode produk                                    |
| msisdn     | Nomor tujuan transaksi                         |
| sn         | Serial Number/kode voucher (jika ada)          |
| msg        | Pesan detail transaksi                         |

---

## Contoh Report

### Transaksi Sukses

```
https://url-mitra.com/report?serverid=T220907KLNU024755&clientid=1541622&statuscode=1&kp=SP10&msisdn=081311223344&sn=02806600001851150526&msg=TRXID:T220907KLNU024755. REFID:1541622. SUKSES, SN:02806600001851150526. HRG:10145. SALDO:126700. KP:SP10.081311223344 TGL:2022-06-21
```

### Transaksi Gagal

```
https://url-mitra.com/report?serverid=T220907KLNU024756&clientid=1541623&statuscode=2&kp=SP10&msisdn=081311223344&sn=&msg=TRXID:T220907KLNU024755. REFID:1541622. GAGAL, Nomor Tujuan Salah. HRG:10145. SALDO:121700. KP:SP10.081311223344 TGL:2022-06-21
```

---

## Tips Implementasi

1. **Cek statuscode** - Gunakan `statuscode=1` untuk sukses dan `statuscode=2` untuk gagal
2. **Simpan serverid** - Gunakan `serverid` sebagai referensi transaksi dari Toko Voucher
3. **Parse msg** - Field `msg` berisi informasi lengkap termasuk harga dan saldo
